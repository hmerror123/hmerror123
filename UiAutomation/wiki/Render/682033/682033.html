There are several rules and recommendation that should help you start using Automation events.
  1. register for as many events as necessary. Events are resource and time consumers.
  2. unregister all or some events as quickly as possibly ( the Unregister-UIAEvent cmdlet).
  3. you can access registered event via the static collection [UIAutomation.CurrentData]::Events (this collection' content is what the Get-UIARegisteredEvent cmdlet returns)
  4. GUI event work on the GUI thread. This means that a range of features tied to the PowerShell thread could became unavailable (for example, the Write-Host cmdlet).
